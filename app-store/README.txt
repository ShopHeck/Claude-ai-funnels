App Store submission — NeuroFunnel AI

Bundle identifier: ai.neurofunnel.app
Display name:      NeuroFunnel
Subtitle:          Immersive funnels, engineered.
Primary category:  Business
Secondary:         Productivity
Age rating:        4+

Required compliance (already wired in this codebase):
- Apple Guideline 4.2 (Minimum Functionality)
  - Native haptics (expo-haptics) on every primary action.
  - Biometric unlock scaffolded (expo-local-authentication).
  - Share sheet + deep linking (expo-linking).
  - Offline-friendly data layer (seed in lib/data/ runs without network).
- Apple Guideline 4.8 (Sign in with Apple)
  - Implemented in components/auth/AppleSignInButton.tsx and wired in sign-in / sign-up.
- Apple Guideline 3.1.1 (In-App Purchase)
  - iOS subscription purchases must use StoreKit 2. Wire RevenueCat in lib/billing/
    behind the existing BillingPort interface — swap mockBilling for a RevenueCat
    adapter before TestFlight. Price parity with the Stripe web prices in lib/billing/plans.ts.
- Apple Guideline 5.1.1 (Privacy)
  - Privacy manifest is generated by app.config.ts (privacyManifests block).
  - NSUserTrackingUsageDescription copy is included; ATT is NOT prompted by default
    (lib/telemetry/posthog.ts is event-only and respects Do Not Track).
- Apple Guideline 5.1.1(v) (Account deletion)
  - Implemented in app/(app)/settings.tsx; calls auth.deleteAccount().

Pre-submission checklist:
1. Replace placeholder icon.png and splash.png in assets/images/ with final artwork
   (1024x1024 PNG, no alpha for icon; 1284x2778 for iPhone 15 Pro Max splash).
2. Set eas.json 'ascAppId' and 'appleTeamId' to your App Store Connect values.
3. Set EXPO_PROJECT_ID in app.config.ts 'extra.eas.projectId'.
4. Configure RevenueCat project and set REVENUECAT_IOS_KEY.
5. Set EXPO_PUBLIC_SUPABASE_URL, EXPO_PUBLIC_SUPABASE_ANON_KEY if wiring Supabase.
6. Run: eas build -p ios --profile production
7. Run: eas submit -p ios --latest
8. In App Store Connect, fill out:
   - App Privacy (email + product-interaction only; no tracking)
   - Pricing (match plans in lib/billing/plans.ts)
   - Screenshots (6.9" and 6.5" iPhone required; 12.9" iPad optional)
   - Review notes: mention that Apple IAP is wired via RevenueCat.

Restricted in the iOS build:
- No external-browser links for paid upgrades (guideline 3.1.3 anti-steering).
- Stripe payments are only reachable from the web target.
